# Fantastic Island v0.3.0

Player source switching and Codex question flow release.

This release broadens the built-in Player module beyond a single now-playing surface and makes Codex prompts more usable inside the island. It also tightens release metadata so the public source tree matches the versioned app build.

## Highlights

- Added Apple Podcasts as a controllable Player source beside Apple Music
- Added an in-artwork source picker for switching between installed Apple playback sources
- Added MediaRemote-backed Podcasts metadata and transport commands, including a persistent Swift helper fallback for now-playing reads
- Improved Player artwork behavior with smoother transitions, current-track fallback artwork loading, and upcoming Apple Music artwork prefetching
- Preserved recent Podcasts metadata briefly when the system now-playing feed drops out
- Reworked Codex structured question prompts into a paged flow for multi-question requests
- Cleared stale Codex approval/question state once later session progress arrives
- Adjusted island auto-presentation to choose the first eligible activity instead of assuming the frontmost activity is always presentable

## Notes

- This is still a `source-only` release.
- Podcasts control uses private MediaRemote symbols in the local app process and a Swift helper fallback. Verify this path on the macOS versions you plan to run.
- The Player source picker currently focuses on installed Apple playback sources; Spotify remains supported by the registry and transport layer but is not part of the default source picker.
- The shared Xcode launch scheme now targets the Release configuration for release verification.

## Build

```bash
xcodebuild -project 'FantasticIsland/FantasticIsland.xcodeproj' -scheme 'FantasticIsland' -configuration Release CODE_SIGNING_ALLOWED=NO build
```

Lightweight logic tests:

```bash
swift test
```
