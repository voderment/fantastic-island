# Fantastic Island v0.2.0

Island transition and interaction polish release.

This release focuses on making the island feel steadier while moving between closed, peek, and expanded states. It also keeps the public repository aligned with the private transition work that is now being continued in open source.

## Highlights

- Reworked island presentation into an explicit transition plan and phase model
- Added frozen render snapshots so peek and expanded content stay stable during morphs
- Locked expanded content layout width during open transitions to avoid right-column drift
- Deferred module publishes and measured-height changes while shell transitions are in flight
- Split module rendering into live hosts and snapshot views for `Codex`, `Clash`, and `Player`
- Fixed collapsed-state hover scaling by routing hover tracking through the closed activation panel
- Improved `Player` refresh behavior with asynchronous AppleScript fetches and artwork throttling

## Notes

- This is still a `source-only` release.
- The animation work is intended as the baseline for future tuning, not the final shape of the shell.
- Window hover and transition feel should be verified on real hardware because these AppKit panel interactions are not covered by the lightweight logic test suite.

## Build

```bash
xcodebuild -project 'FantasticIsland/FantasticIsland.xcodeproj' -scheme 'FantasticIsland' -configuration Debug CODE_SIGNING_ALLOWED=NO build
```

Lightweight logic tests:

```bash
swift test
```
